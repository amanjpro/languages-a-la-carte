  11. trait NamerComponent extends TransformationComponent[Tree, Tree]

    Namer binds name uses to their definitions.

    Namer binds name uses to their definitions.

    This phase reports bind-errors and attaches symbols of the definition trees to the use-trees. This phase expects all definition trees to have their symbols attached to.

    This phase only assigns names to the top-level definitions and does not go all the way down to the trees. For example it assigns names to the methods, and variable definitions defined as top-level trees, but it never assigns names to their right-hand-side trees (their bodies).
